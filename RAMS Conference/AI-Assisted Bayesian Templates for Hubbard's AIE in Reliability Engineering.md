#### **Full Abstract** (478 words)

Reliability, maintainability, and safety engineers routinely face decisions under uncertainty—sparse failure data, variable costs, ambiguous test results, and complex system interactions. Doug Hubbard’s Applied Information Economics (AIE), as presented in *How to Measure Anything*, offers a proven quantitative framework that integrates calibrated expert estimates, Bayesian updating, Monte Carlo simulation, and Value of Information (VoI) analysis. However, two persistent barriers block widespread adoption: (1) creating high-quality probability distributions from limited data or judgment, and (2) determining which pieces of information matter and exactly where they fit into the Bayesian model (as priors, likelihoods, evidence, or posteriors).

This practical 2-hour tutorial removes those barriers using ready-to-use workflow templates (Excel for accessibility, Jupyter Notebooks for scalability) that work seamlessly with any large language model (LLM) or AI assistant. The central innovation is an AI-assisted process that takes a plain-English problem description plus any available sample data and returns a clean JSON structure. This JSON breaks the problem into modular “measurement blocks,” each clearly specifying:

- Key uncertainties and decision variables  
- Appropriate prior distributions (seeded by calibrated estimates or fitted from data)  
- Likelihood models for new evidence  
- Exact placement in the Bayesian chain or network  
- Monte Carlo parameters for propagation and VoI calculations  

The AI also supplies plain-language explanations for every step: why specific data belongs in a particular block, how the Bayesian update formula applies, the rationale for the chosen distribution (e.g., Beta for binomial reliability, Weibull for time-to-failure), and how to reduce bias through calibrated estimation.

**Tutorial Format (Fully Hands-On):**  
- **20 min** – Concise overview of AIE principles with relevant R&M examples (failure rate updating, maintenance optimization, risk ranking).  
- **40 min** – Live demonstration of effective prompting techniques that produce high-quality JSON outputs, followed by end-to-end template workflow.  
- **40 min** – Guided participant exercises on real R&M scenarios (e.g., updating component reliability with limited tests; deciding on additional inspections via Expected Value of Perfect Information).  
- **20 min** – Open discussion on validation, scaling to complex systems, common pitfalls, and extensions (digital thread integration, batch processing).

All attendees receive the complete template package, prompt library, example JSON mappings, and step-by-step guides. No advanced statistics or programming background is required. By session end, participants can immediately apply the method to their own problems: feed a description into an LLM, receive structured Bayesian blocks, load into templates, run simulations, and generate auditable recommendations.

The approach directly solves the historical confusion around “Which information is valuable?” and “How do I build the right distribution?” The AI handles the structural mapping and explanation layers, while the templates manage calculations and visualizations. This combination makes Hubbard’s AIE accessible, repeatable, and fast—turning uncertainty quantification into a practical daily tool for reliability professionals.

The tutorial aligns with RAMS priorities in risk analysis, AI/ML applications, and practical decision support under uncertainty. It demonstrates how AI-augmented templates overcome long-standing implementation challenges, enabling anyone to engineer robust Bayesian solutions from problem descriptions and data without getting lost in the details.
